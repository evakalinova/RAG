Ollama — Lokální modely

Všechny modely běží na firemním serveru · žádná data neodcházejí · žádné API poplatky
🤖
Mistral 7B — hlavní model
ollama pull mistral
Generuje odpovědi uživatelům, vytahuje metadata z dokumentů, píše shrnutí. Vyžaduje GPU s 8GB VRAM (RTX 3070) nebo běží pomaleji na CPU.
mistral:latestlangchain-ollamaChatOllama
GPU 8GB VRAM (doporučeno)
🔢
nomic-embed-text — embeddingy
ollama pull nomic-embed-text
Převádí text na vektory čísel — musí být použit stejný model při ingestu i při dotazech. Specializovaný embedding model, lehčí než Mistral 7B.
nomic-embed-textlangchain-ollamaOllamaEmbeddings
CPU stačí · nenáročný
🚀
Mixtral 8x7B — upgrade
ollama pull mixtral
Silnější model pro lepší češtinu a složitější dotazy. Výměna = změna jednoho řádku kódu díky LangChain. Vyžaduje více výkonu.
mixtral:latestlangchain-ollamaChatOllama
GPU 24GB VRAM · 2. iterace

Ingest Pipeline

Běží na pozadí · admin spouští
⚡
FastAPI — endpoint /ingest/sync
Admin zavolá endpoint → FastAPI zapíše úlohu do Redisu → Celery ji vyzvedne a spustí pipeline na pozadí → FastAPI okamžitě vrátí "spuštěno".
fastapiceleryredis
↓
ingest.py — orchestrátor · volá kroky 01–09 v pořadí, předává výstupy
01
📁
SharePoint Connector
sharepoint.py
Připojí se na SharePoint REST API, projde složky. Porovná datum modifikace + SHA-256 hash (jedinečný otisk obsahu — pokud se hash nezměnil, soubor je identický). Stáhne pouze nové nebo změněné PDF do data/raw/.
office365-rest-python-clientrequestshashlib (stdlib)os (stdlib)
↓ přihlašovací údaje z .env
↑ PDF soubory v data/raw/
02
📄
PDF Parser
pdf_parser.py
Převede PDF na čistý Markdown lokálně — žádná data neodcházejí. pymupdf4llm zachovává tabulky, nadpisy a strukturu stejně jako LlamaParse, ale běží 100% offline. Proč ne OCR? Směrnice jsou digitální PDF, ne skeny — OCR je pro papírové dokumenty.
pymupdf4llm (lokální)pip install pymupdf4llm
↓ PDF soubor
↑ Markdown text
03
🧹
Text Cleaner
cleaner.py
RegEx vzory odstraní šum: záhlaví, zápatí, čísla stran, právní doložky. RegEx místo AI — deterministické, levné, rychlé, 100% předvídatelné. Žádná data neodcházejí.
re (stdlib — bez instalace)
↓ Markdown + šum
↑ čistý text
04
🏷️
Metadata Agent
metadata_agent.py
Ollama (Mistral 7B) přečte začátek dokumentu a vytáhne: název, oddělení, platnost od/do, verze, pro koho platí. Výstup validuje Pydantic. Vše běží lokálně — žádná data neodcházejí.
Ollama · mistral:latestlangchain-ollamaChatOllamapydantic
↓ čistý text
↑ JSON s metadaty
05
📝
Summary Agent
summary_agent.py
Ollama (Mistral 7B) vygeneruje 1–2 větné shrnutí dokumentu. Pomáhá rerankeru správně seřadit výsledky a uživateli pochopit z jakého dokumentu odpověď pochází. Lokálně.
Ollama · mistral:latestlangchain-ollamaChatOllama
↓ čistý text
↑ shrnutí (1–2 věty)
06
✂️
Text Chunker
chunker.py
Rozdělí text na bloky ~1000 znaků s přesahem 150 znaků. Model nedokáže přečíst 100 stran najednou. Přesah zajistí, že žádná věta nebude rozseknuta bez kontextu.
langchainRecursiveCharacterTextSplitter
↓ čistý text
↑ seznam chunků
07
📦
Document Builder
chunker.py
Každý chunk + metadata + shrnutí se zabalí do jednoho objektu. Metadata jsou přilepena k textu a cestují s ním až do Qdrantu a zpátky k uživateli jako citace.
langchainDocument objekt
↓ chunk + metadata + shrnutí
↑ Document objekt
08
🔢
Embedder
embedder.py
Převede text chunku na vektor čísel. Stejný model musí být použit i pro embeddingy dotazů — jiný model = jiná čísla = vyhledávání nefunguje. Kritické pravidlo. Vše lokálně.
Ollama · nomic-embed-textlangchain-ollamaOllamaEmbeddings
↓ text chunku
↑ vektor čísel

💾
Indexer → Qdrant

Uloží balíček do Qdrantu: vektor + původní text + metadata. Qdrant je připraven odpovídat na dotazy v milisekundách. Zapíše záznam do SQLite (hash, datum, počet chunků).
qdrant-clientlangchain-qdrantsqlite3 (záznam dokumentu)

↑ uloženo v Qdrantu + SQLite ✓

Chat Pipeline

Spouští se na každý dotaz uživatele
⚡
FastAPI — endpoint /chat
Uživatel pošle dotaz → FastAPI ho přijme → synchronně spustí celou chat pipeline → vrátí odpověď přímo. Na rozdíl od ingestu zde Celery není — chat musí být rychlý.
fastapiuvicornpydantic

01
🔢
Query Embedder
rag_chain.py
Otázka uživatele se převede na vektor — stejným modelem jako dokumenty při ingestu (nomic-embed-text). Jiný model = jiná čísla = nefunkční vyhledávání. Kritické pravidlo.

🔍
Hybrid Retrieval
rozšíření · rag_chain.py
Kombinuje vektorové hledání (sémantika) + BM25 fulltext (přesná slova). Zachytí obecné dotazy i firemní zkratky (BOZP, TPP, §52). Filtruje i podle metadat. Qdrant běží lokálně.
qdrant-clientlangchain-qdrantBM25 + vector (nativní v Qdrantu)

↑ top 10 chunků

🎯
Reranker
rozšíření · reranker.py
Projde 10 kandidátů a seřadí je podle skutečné relevance. Eliminuje falešně pozitivní výsledky. Open-source, běží 100% lokálně — žádná data neodcházejí, žádné poplatky.
BGE-Reranker-v2-m3 (lokální)FlagEmbedding

↑ top 3–5 seřazených

🤖

↑ text odpovědi
